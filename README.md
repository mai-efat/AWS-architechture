🌐 High-Availability AWS InfrastructureThis repository contains the architecture design and deployment strategy for a robust, scalable, and secure cloud environment on Amazon Web Services (AWS).🏗️ Architecture OverviewThe system follows a Multi-Tier Architecture designed for high availability and fault tolerance across multiple Availability Zones.🛡️ Networking & SegmentationThe infrastructure is contained within a VPC with a CIDR block of 10.0.0.0/16. It is logically segmented into:Public Subnets: Host internet-facing components (e.g., 10.0.1.0/24).Private Subnets (App Tier): Securely host the application logic using Auto Scaling groups.Private Subnets (Data Tier): Deeply isolated layers for database management (e.g., 10.0.4.0/24).⚡ Key FeaturesAuto Scaling: Uses Auto Scaling groups  to dynamically adjust capacity based on traffic, ensuring performance and cost-efficiency.DNS Management: Integrated with Route 53  for highly available and scalable domain name routing.Security Groups: Multi-layered firewalls (Security Groups)  are applied at every tier to strictly control traffic flow.Database Reliability: Implements RDS Multi-AZ  to provide automatic failover and data redundancy across different zones.🛠️ Tech StackCloud Provider: AWSNetworking: VPC, Route 53, Subnetting (Public/Private).Compute: Auto Scaling Groups.Database: RDS Multi-AZ.
